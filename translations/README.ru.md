# Перевод AI Agents from Scratch на русский язык

## Структура

```
translations/
├── README.ru.md              ← этот файл
├── GLOSSARY.ru.md            ← глоссарий терминов (ОБЯЗАТЕЛЕН к использованию)
├── manifest.json             ← трекинг оригиналов и статусов перевода
└── ru/
    ├── 01_intro/             ← переводы уроков
    │   ├── CONCEPT.md        ← русская версия concepts
    │   └── CODE.md           ← русская версия code walkthrough
    ├── 02_openai-intro/
    └── ...
```

## Источники

Оригинальные тексты лежат в `examples/`:

```
examples/
├── 01_intro/
│   ├── CONCEPT.md            ← оригинал (EN)
│   ├── CODE.md               ← оригинал (EN)
│   └── intro.js              ← код (не переводится)
└── ...
```

Корневые файлы для перевода:
- `README.md` → `README.ru.md` (в корне репо)
- `CONTRIBUTING.md` → `CONTRIBUTING.ru.md`
- `DOWNLOAD.md` → `DOWNLOAD.ru.md`
- `PROMPTING.md` → `PROMPTING.ru.md`

## Правила перевода

### Обязательно

1. **Глоссарий**: все термины — по `GLOSSARY.ru.md`. Новый термин? Сначала утвердить, потом переводить.
2. **Код-блоки**: БЛОКИ КОДА (```bash, ```python, ```javascript, ```mermaid, ```text) — **НЕ ТРОГАТЬ**. Даже комментарии внутри кода остаются на английском.
3. **Ссылки**: сохранять оригинальные ссылки. Локальные пути не переводить.
4. **Форматирование**: Markdown-структура (заголовки, списки, таблицы, блоки кода) — идентична оригиналу.
5. **Тон**: учебный, дружелюбный. Вы — «Вы» (с заглавной).

### Запрещено

- Переводить код внутри блоков (включая комментарии bash/python)
- Заменять технические термины кальками («отправляй в remote» → «отправляй на сервер»)
- Добавлять/убирать разделы — перевод должен иметь ту же структуру
- Переводить названия библиотек, моделей, протоколов

### Рекомендации

- Переводить `CONCEPT.md` (концепции) до `CODE.md` (разбор кода) — так понятнее контекст
- При неоднозначности — отдавать предпочтение устоявшемуся переводу
- Проверять длину абзацев: русский текст обычно длиннее английского на 20-30%

## Трекинг изменений

`manifest.json` содержит SHA-256 хеши оригинальных файлов. Если автор обновит оригинал:

1. Хеш в `manifest.json` не совпадёт с текущим файлом
2. Статус файла автоматически помечается как `stale`
3. Запускается re-translation только для изменённых файлов

## Pipeline перевода

```
1. structure-scanner    → 扫描 файлов
2. glossary-manager     → глоссарий
3. markdown-translator  → перевод
4. block-verify         → проверка код-блоков
5. technical-reviewer   → ревью
6. russian-editor       → редактура (только при REQUEST_CHANGES)
7. format-checker       → формат
8. diff-reviewer        → diff
9. commit               → коммит
```

## Контрибьюторы

Если хотите помочь с переводом:
1. Ознакомьтесь с `GLOSSARY.ru.md`
2. Выберите файл из `manifest.json` со статусом `pending`
3. Переведите по правилам выше
4. Создайте PR с указанием переведённых файлов
